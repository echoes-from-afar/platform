# Security Policy

## Supported Versions

Security fixes are applied to the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

1. Open a private security advisory in GitHub if available for this repository.
2. If advisory flow is unavailable, open an issue requesting a private channel and avoid sharing exploit details publicly.
3. Include:
   - affected files/components
   - reproduction steps
   - potential impact
   - suggested mitigation (if known)

## Response Expectations

- Initial acknowledgment target: within 72 hours.
- Triage and risk classification target: within 7 days.
- Fix timeline depends on severity and maintainers' capacity.

## Disclosure

Once a fix is available, maintainers will document:

- the affected area
- the patch release/commit
- any required user action
